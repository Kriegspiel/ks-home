# Website Track IA + Contracts (Slice 910)

## Canonical Routes

- `/` home
- `/leaderboard` leaderboard landing
- `/blog` blog index
- `/blog/:slug` blog detail
- `/changelog` changelog index
- `/changelog/:slug` changelog detail
- `/rules` rules index

## Route Ownership Matrix

- IA + routing contracts: `ks-home`
- Public content documents (blog/changelog/rules markdown): `content`
- Step/slice planning and acceptance evidence: `ks-platform`

## URL Deprecation and Redirect Policy

- Permanent migrations use `308` when successor route exists.
- Removed endpoints with no replacement return `410`.
- New URLs must be additive/non-breaking unless redirect policy is updated and tested.

## Navigation and Fallback Behavior

- Global nav includes Home, Leaderboard, Blog, Changelog, Rules.
- Broken/unknown internal routes return `404` via fallback page `/404`.
- Explicitly retired endpoints may return `410`.

## Content Contract

`Kriegspiel/content` is the source of truth for `blog`, `changelog`, and `rules` markdown collections.

Required frontmatter fields:

- `title`
- `slug`
- `summary`
- `publishedAt`
- `updatedAt`
- `author`
- `tags`
- `draft`
- `version` (required for changelog/rules)

## Static Generation Foundation

`ks-home` consumes `../content` markdown and generates static output under `dist/`.

- Collection index pages: `/blog`, `/changelog`, `/rules`
- Detail pages generated by slug for blog/changelog
- Route crawl in tests ensures no broken internal links from generated pages.

## Baseline Budgets (applies to slices 920/930/950/960)

- LCP <= 2.5s p75
- CLS <= 0.1
- Initial route JS <= 220KB gz

## Accessibility Floor (WCAG 2.2 AA baseline)

- Keyboard navigation required on all primary routes
- Landmark structure (`header`, `main`, `footer`) present
- Heading order consistency enforced in templates
- Color contrast must satisfy WCAG 2.2 AA thresholds
